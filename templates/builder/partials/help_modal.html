How to Use Code2Resume

A practical guide to getting the best results

{# ---- Workspace ---- #}

Workspace & Editor

Split-Pane Layout HTML and CSS editors on the left, live preview iframe on the right. The preview updates on every keystroke — no save button needed.
Code Editors Powered by CodeJar and Highlight.js. Tab inserts two spaces, Ctrl+Z undoes, Ctrl+Y redoes. Standard browser shortcuts (Ctrl+A, Ctrl+C, Ctrl+V) work normally inside the editors.
Auto-Save Your work saves to your session every 1.5 seconds automatically. The "Saved / Saving…" indicator in the navbar reflects this. Refreshing or reopening the tab restores your last state. Logged-in users can also save named resumes to their profile with the Save button.
{# ---- Framework & CSS ---- #}

Framework Injector — What to Expect

None (plain CSS) Most reliable for PDF output. You have complete control over every pixel. Use pt or mm units for font sizes and spacing when targeting print — they map directly to the PDF output with no surprises.
Tailwind CSS The Play CDN runs a JavaScript engine that scans your HTML and generates styles on the fly. It works well in the preview and in PDF output. Avoid @apply — it requires a build step and won't work via CDN. Stick to utility classes directly on elements. Responsive variants (sm:, md:) are available but irrelevant in PDFs since PDFs have no viewport.
Bootstrap Bootstrap's CSS resets (box-sizing, margin, font) apply globally inside the preview and PDF. This can shift your custom CSS in unexpected ways. If something looks off, inspect which Bootstrap rule is overriding yours and add a more specific selector. Bootstrap's grid system works, but PDF pages have no concept of viewport breakpoints — your layout will always render at the paper width.
Mixing frameworks with custom CSS Your CSS in the CSS tab is loaded after the framework, so your rules take precedence. Use this to override framework defaults. If a utility class is fighting your custom style, add !important sparingly or increase selector specificity.
{# ---- Page break guidelines ---- #}

Page Break Guides

What the guides show The dashed red lines mark where Playwright will cut the page when generating the PDF. Content that crosses a line will be split across two pages. Toggle them with the Guides button in the navbar.
Keeping content on one page If a section is cut mid-way, reduce font size, line-height, or section padding until the content fits above the guide line. A font size of 13–14px with 1.4–1.5 line-height is a good starting point for a dense single-page resume. You can also use page-break-before: always or page-break-inside: avoid in your CSS to control where breaks happen.
Why the guides account for margins The PDF is generated with 10mm margins on all sides. The guide positions account for this — they mark the printable content area, not the full paper height. Content between guides is safe; content that crosses a guide will be clipped or pushed to the next page.
{# ---- PDF best practices ---- #}

PDF Output — Best Practices

Fonts Use web-safe fonts (Georgia, Times New Roman, Arial, Segoe UI) or Google Fonts loaded via a <link> tag in your HTML. Custom fonts load correctly during PDF generation because Playwright fetches external resources. Avoid base64-embedded fonts — they inflate the resume HTML and slow down generation.
Images and photos Images linked via URL render correctly in the PDF as long as the URL is publicly accessible. Use absolute URLs (starting with https://). Locally hosted images will not render in the PDF since Playwright runs server-side and cannot access your local machine.
Colours and backgrounds The PDF engine has print_background: True enabled, so background colours and images print correctly. If a background colour is missing from your PDF, add -webkit-print-color-adjust: exact to the relevant element.
Sizing units The resume renders at 96dpi in a browser context. px units behave exactly as they do in the preview. rem and em work normally. Avoid vw and vh — the PDF viewport is the paper width, not the browser window, so these units produce unpredictable results.
Keeping it to one page Watch the first page break guide. If your content reaches it, try: reducing line-height from 1.6 to 1.4, trimming margin-bottom on section headings, reducing body padding, or dropping font size by 1px. Small adjustments compound across a full page.
What the preview doesn't show The preview is a browser iframe — it renders identically to how Playwright will render your resume, with one exception: print-specific CSS (@media print) is not active in the preview but will be active in the PDF. Use this deliberately to hide elements in the PDF that are visible in the preview, or vice versa.
{# ---- Paper sizes ---- #}

Paper Sizes

A4 (794×1123px) — Standard in most of the world. Use this unless you're applying to North American companies.
Letter (816×1056px) — Standard in the US and Canada. Slightly wider and shorter than A4.
A5 (559×794px) — Half the size of A4. Useful for concise one-page summaries or compact portfolio cards.
Legal (816×1344px) — Taller than Letter. Rarely used for resumes but useful for very detailed CVs.
